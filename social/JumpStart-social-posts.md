## LinkedIn

**Image:** `jumpstart_hero_landscape.png`

I got tired of the daily ritual of `cd` into five folders, running the same start commands, hunting for which port each thing grabbed, and killing zombie processes when something wouldn't die.

So I built JumpStart.

It's a native macOS control panel for every app and project on your machine. Add a project once, let it auto-detect the runnable parts (Node, Go, Python, Rust, Docker Compose, and more), then start, stop, and watch everything from one window.

What it does:
- One-click start/stop for a single process or a whole project
- Live status: PID, exit code, logs, CPU and memory per process
- Automatic port detection with clickable localhost badges
- Per-process env vars and .env import
- A per-project Kanban board with an AI assistant that drafts stories and tasks using a local Ollama model (your code and notes never leave your machine)

If you're a solo founder, freelancer, or someone juggling side projects, this is the dashboard I always wanted: less terminal juggling, more building.

It's live and free to try: [LINK]

Would love feedback from anyone running more than one project locally. What would you want it to do?

So I made JumpStart. You point it at a project folder and it auto-detects the runnable parts (Node, Go, Python, Ruby, PHP, Java/Maven, Gradle, Rust, Docker Compose), then you start/stop them individually or all at once.
